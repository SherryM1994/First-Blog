由零开始创建个人博客3（Git与GitHub的绑定）

使用Git就是为了远程管理已有的项目，不需要随身携带处理过项目的主机，实现随时随地可以修改数据。将Git与GitHub绑定，就可以将GitHub上的文件clone到Git本地上，实现实时管理。

要用git上传文件到GitHub首先得利用SSH登录远程主机

一、利用SSH将GitHub与Git绑定

1、打开Git Bash窗口，输入ssh\检测是否安装了ssh

上图结果显示已经安装了ssh，进行下一步

2、输入 ssh-keygen -t rsa 命令，表示我们指定 RSA 算法生成密钥，然后敲四次回车键\，之后就就会生成两个文件，分别为秘钥 id_rsa 和公钥 id_rsa.pub. （注意：git中的复制粘贴是 Ctrl+insert 和 Shift+insert）\文件的位置在 Git Bash 上面都有显示，默认生成在以下目录：

  • Linux 系统：~/.ssh
  • Mac 系统：~/.ssh
  • Windows 10 ：/c/Users/Administrator/.ssh

3、接下来我们要做的就是把公钥 id_rsa.pub\ 的内容添加到 GitHub。复制公钥 id_rsa.pub 文件里的内容，你可以通过目录找到 id_rsa.pub 文件的位置，用记事本打开并复制。也可以使用命令直接在 Git Bash 上打开再Ctrl+insert （推荐）：

4、得到公钥后，进入GitHub主页，点击右上角的settings

5、点击SSH and GPG keys，在点击New SSH key

6、将在Git Bash里复制的公钥shift insert到key的框框里，然后add SSH key

接下来会弹出确认对话框，将密码填入并confirm

7、最后去 Git Bash 确认一下是否真的验证成功

我们打开Git Bash 输入 ssh -T git@github.com\ 弹出以下对话框，将 yes\ 填入即可

然后弹出以下结果，很明显已经绑定成功

三、上传文件

1、提交文件

提交文件的两种方法：

第一种：本地没有 git 仓库

  1. 直接将远程仓库 clone 到本地；
  2. 将文件添加并 commit 到本地仓库；
  3. 将本地仓库的内容push到远程仓库。

第二种：本地有 Git 仓库，并且我们已经进行了多次 commit 操作

  1. 建立一个本地仓库进入，init 初始化；
  2. 关联远程仓库；
  3. 同步远程仓库和本地仓库；
  4. 将文件添加提交到本地仓库；
  5. 将本地仓库的内容 push 到远程仓库。

而我是首次安装Git，本地是没有Git仓库的，所以我选择第一种方法

1.1 进入GitHub个人主页，将新建的repository上传到Git

1.2 点击项目进入界面，点击clone or download，然后复制地址

1.3 在打算储存 Git仓库 的文件夹中打开 Git Bash，我实现在D盘新建了一个 Git 文件夹

可以通过cd命令访问该文件夹

4 进入我们准备存储 Git 仓库的目录，输入git clone+在GitHub中复制的地址SherryM1994/SherryM1994.github.io ，如下图：

2.1 首先在本地仓库目录下新建一个demo文件夹

2.2 使用git init 初始化仓库

2.3 使用命令 git remote add origin SherryM1994/SherryM1994.github.io 关联远程仓库

再输入命令 git pull origin master 将远程仓库与本地仓库同步
